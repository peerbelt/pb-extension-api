Initializing Peerbelt

It is extremely easy to detect and work with Peerbelt. The above box displays the result of running just few lines of code.

Whenever Peerbelt is available and enabled in the browser, the API will pass true into the callback defined on line 6. At this point, you can start making additional calls to Peerbelt.

Notice the first parameter in the init() call. While an empty object will allow Peerbelt detection, to make privileged calls you must pass in a Peerbelt issued application identifier. The identifier tells Peerbelt what domains are owned by you. This does protect the user's privacy by not exposing browsing data bound to somebody else's Web domain.

<script type="text/javascript" src="pb.0.0.3.js" charset="utf-8" ></script>
<script>
(function() {
    window.onload = function(){ 
        var api = window["pbAPI"]; if (!api) { return; } 
        api.init({}, function(e){
    	    if (e) {
    		   // Peerbelt is here
    		   document.getElementById("instrument").className = "available"; } } ); } } )();
</script>

Requesting Application Identifiers

Tweet @peerbelt with #pb-api-request to get an application identifier. We queued updates to both the API and client libraries to make the environment more secure while giving you access to interesting data that helps you better serve your users.
